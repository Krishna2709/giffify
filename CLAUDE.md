## What this is

A single portable **Agent Skill** named `video-to-gif` (open Agent Skills format: `SKILL.md` + `scripts/` + `references/` + `assets/`) that converts explicit timestamp ranges from local videos into optimized GIFs. One shared implementation, packaged into two thin distribution plugins:

- Claude Code plugin (`.claude-plugin/plugin.json`)
- Codex plugin (`.codex-plugin/plugin.json`)

The canonical skill source lives at `src/skill/video-to-gif/` (per spec §5); platform packages under `packages/` are generated by a build script, never edited directly.

### Architecture layers (spec §4)

1. **Agent layer** (Claude Code / Codex): interprets natural language, asks only required questions, obtains approvals (installs, overwrites, external paths), summarizes results.
2. **Python engine** (`scripts/video_to_gif.py` + `vtg/` package): deterministic, **non-interactive** — never prompts. Parses timestamps/manifests, validates, plans outputs, runs FFmpeg, reports structured results.
3. **FFmpeg/ffprobe**: ffprobe inspects media (JSON output); FFmpeg encodes GIFs via two-pass palette generation (`palettegen` → `paletteuse`).

### CLI contract (spec §12)

```
python scripts/video_to_gif.py doctor --json
python scripts/video_to_gif.py inspect --input <video> --json
python scripts/video_to_gif.py create --input <video> --start 00:01:00 --end 00:01:05 --profile balanced --json
python scripts/video_to_gif.py batch --manifest clips.json [--dry-run] --json
python scripts/video_to_gif.py validate-config --config .video-to-gif.json --json
python scripts/video_to_gif.py validate-manifest --manifest clips.json --json
```

With `--json`: final JSON document on stdout, progress as JSON Lines on stderr. Exit codes 0–13 are defined in spec §14 and are part of the contract (there is deliberately no exit code 1). `create` also accepts `--output-name` (bare filename, no path separators), `--invalid-timestamp-policy {fail,skip,clamp}`, `--config`, `--loop`, `--width`, `--fps`, `--colors`, `--allow-upscale`, `--allow-outside-project`.

## Hard constraints from the spec

These are normative (MUST-level) and apply to any implementation code:

- Python 3.10+, **standard library only** for the core runtime (test deps may be optional dev groups). `pip install ffmpeg` is not FFmpeg — detect the real `ffmpeg`/`ffprobe` executables.
- **Never `shell=True`**; subprocess args as arrays; manifest/config values are data only (no env expansion, no hooks, no expressions).
- Timestamps normalized internally to **integer milliseconds**. Accepted forms: `75`, `75.5`, `MM:SS`, `HH:MM:SS`, with optional `.mmm`. A clip takes `start` + exactly one of `end`/`duration`.
- Default output dir `./output`; default collision policy **fail** (engine never overwrites); writes outside project root require explicit authorization.
- Invalid timestamps detected in preflight, before any encoding; policies are `fail` (default) / `skip` / `clamp`.
- Batch: `continueOnError` defaults true; one clip's runtime failure must not stop the rest; result distinguishes created/failed/skipped per clip.
- FFmpeg writes to a temp path first; destination appears only via atomic move after verification. Cancellation must clean temp/palette/partial files but preserve completed GIFs.
- v0.1.0 performs **no network access** — URLs return `UNSUPPORTED_REMOTE_SOURCE` (remote sources are v0.2.0). Enforced at the FFmpeg layer (SEC-010): invoke ffmpeg/ffprobe with `-protocol_whitelist file,pipe` and reject reference-following containers (HLS/DASH/concat → `UNSUPPORTED_MEDIA_CONTAINER`, exit 5).
- Resource limits (SEC-011): per-clip wall-clock timeout + temp-disk ceiling, configurable via `limits` in config (defaults 600s / 2 GiB), exceeded → `RESOURCE_LIMIT_EXCEEDED`, exit 13.
- Project config file is `.video-to-gif.json`; precedence: CLI arg > request instruction > project config > built-in default.
- Quality profiles: small (480px/10fps/128 colors), balanced (640px/15fps/256), high (960px/20fps/256), custom. Widths are maximums; preserve aspect ratio; no upscaling by default.
- Backward compat: patch releases must not break CLI flags or schemaVersion-1 config/manifest/result schemas.

## Testing (spec §22)

Four suites (217+ tests, stdlib `unittest`): `tests/unit` (hermetic, subprocess mocked), `tests/integration` (real ffmpeg against synthetic media from `tools/generate_test_video.py`), `tests/security` (§22.4), `tests/acceptance` (AC-001..014 one test per criterion; AC-015 is a manual procedure in `tests/acceptance/AC-015-manual.md`). Shared fixtures in `tests/fixtures/base.py` create a fake temp project root so all engine writes land outside the repo. Integration tests generate synthetic media at test time (never commit sample videos); unit tests cover timestamp parsing, validation edges, manifest parsing, filename sanitization (Windows reserved names, Unicode), collision policies, and project-boundary checks. Security tests must prove manifest values can't execute commands, filenames can't escape the output directory, hostile playlist files can't trigger network access, and resource limits clean up after themselves. Timestamp/manifest parsers need seeded generative fuzz tests (spec §22.5). Test framework: stdlib `unittest` (zero dependencies) — run with `python3 -m unittest discover -s tests/unit -v`.
